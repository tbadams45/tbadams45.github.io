How to Batch Convert Text Files to Word Documents

Basic steps from the video

  • Generate the zip file containing the text files using the parser at tbadams.com/bercot.
  • Extract the text files into a specific folder. Here, I'm going to assume you've extracted them to a folder in your C drive called "parser". Therefore, the complete path name for the folder would be "C:\parser", but it could be anywhere on your computer.
  • Go to https://gallery.technet.microsoft.com/office/Batch-convert-txt-files-to-d7f53cb9.
  • Download "ConvertTXTFileToWordDocument (Powershell).zip".
  • Extract this zip file to a folder called "txt-to-word" in your "C:\" folder.
  • Open Windows Powershell as an administrator.
  • Execute the following commands in Powershell. Notes: Type "y" when prompted after the first command. Type "r" when prompted after the third command. The path name in the 4th command needs to match where you stored the text files.
    • Set-ExecutionPolicy -Scope Process Unrestricted
    • cd C:\txt-to-word
    • Import-Module .\ConvertTXTFileToWordDocument.psm1
    • ConvertTo-OSCWord -Path C:\parser
  • Open your word documents, which will be located in the same folder as the text files were. (Tip: You can press "Ctrl+A" to select the whole document if you want to quickly change the font and font size.)

    • Once you've done this once, you can skip steps 3, 4, and 5.
