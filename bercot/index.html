Notes

If the program doesn't seem to be generating any useful output, the first thing to check is your use of curly braces ("{" and "}"). There should only be one "{" and "}" per quote, and they should only be used to surround the scripture references. For example, here are some correct usages of curly braces:

  • {2 Cor 5:14}
    "This is the actual quote." Origen
  • {Gen 1:1; Jn 1:1; Creation}
    "This also works."
    -Clement of Rome
Here are some incorrect usages:
  • {{2 Cor 5:14}}
    "This fails because you use two curly braces on each side instead of one." Origen
  • {Gen 1:1; Jn 1:1; Creation}
    "This fails because I put a curly brace in {the middle of} the quote."
    -Clement of Rome

If the program generates useful output but has some trouble understanding a particular quote, it'll tell you below under the "progress" header. Check for the following:

  • You need to use semicolons (;) to separate the scripture references, instead of colons (:) or commas.
  • Is the book abbreviation you're using the correct one? If not, quotes from the same book of the Bible will appear in different PDF files. Take a look at the approved abbreviations for each book.
  • Did you include a reference line but didn't include any quote body beneath it? (e.g. "{Beard}" followed immediately by another reference such as "{2 Cor 3:14}", with no quote in between.) This is especially likely if you don't see an actual quote after the "I had some trouble understanding this reference" body.

If you find you have a ".pdf" file in your output (that is, a file with just the extension but no name), that means in at least one place you had an extra semicolon in your scripture references, e.g. {Rev 5:8; ; Jn 12:3}
